What is merchant-mcp?

merchant-mcp is a protocol-neutral translation layer for AI-driven commerce. Merchants integrate once — the server exposes their catalog and checkout to any AI agent protocol: MCP, OpenAI SDK, UCP, Visa TAP, and others.

New protocols are added as adapters. When the ecosystem evolves, merchant-mcp adapts. Your integration stays the same.

Three tools. Complete commerce.

Tool
get_catalog

Retrieve products available from a merchant. Returns name, description, price, currency, availability, and SKU.

{
  "merchant_id": "demoshop",
  "category": "electronics",
  "limit": 5
}
Tool
create_checkout_session

Initiate a payment checkout session. Returns a checkout URL and session ID.

{
  "merchant_id": "demoshop",
  "items": [
    {
      "sku": "ELEC-001",
      "quantity": 1
    }
  ],
  "currency": "PLN"
}
Tool
get_transaction_status

Check the current status of a checkout session. Returns status, amount, and payment timestamp if completed.

How it works with your catalog

merchant-mcp is a protocol layer, not a product catalog. It defines how AI agents communicate with merchants — what they can ask, and what they receive back. Your catalog, inventory, and checkout logic live in your own systems. merchant-mcp connects them to any AI agent through a standardized adapter.

01

You bring your catalog

Your products, prices, and inventory stay in your systems — an e-commerce platform, a PIM, or a custom database.

02

You build an adapter

An adapter connects your catalog to the MerchantAdapter interface. One adapter, any AI agent protocol.

03

AI agents can shop

Any MCP-compatible agent can discover products, initiate checkout, and check payment status — without custom integration per platform.

The DemoShop adapter included in this repository is a working reference implementation. Replace it with your own adapter to connect a real merchant catalog.
